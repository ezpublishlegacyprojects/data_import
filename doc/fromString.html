Authors

data type string

ezauthor

Expected input

The authors list, separated by '&'. For each author, name email and ID have to be provided, separated by pipes (|). ID can be a numerical value. If -1 is given, the ID will be automatic.

Example

Input

john Doe|john@doe.com|-1&Jane Doe|jane@doe.com|-1

Result

Two authors in the attribute (John Doe, email address john@doe.com, and Jane Doe (jane@doe.com).

The rows list, separated by &, and for each row, the cells list, separated by a pipe. The matrix class attribute has to match the input data (number of cells). Number of rows is automatic.

Multi option

data type string

ezmultioption

Expected input

First separator is &. The first value is the multioption name, and each other '&' separated string is a specific option.

For each option, elements are separated by a pipe. The first item is the option name, the second item the default value, and the next items are option values, with each time the option name AND the option price:

<MultiOptionName>& <Option1Name>|<Option1DefaultValue>|<Option1Value1>|<Option1Price1>|<Option1Value2>|<Option1Price2>& <Option2Name>|<Option2DefaultValue>|<Option2Value1Name>|<Option2Value1Price>|<Option2Value2Name>|<Option2Value2Price>

Example

Input

Cloth option&Color|Red|Red|0|Green|0|Blue|0&Size|M|S|0|M|0|L|2|XL|5|XXL|10

Result

A multi-option named « Cloth options » with two options: Color and Size.

Color will have a default value of « Red », and possible options will be Red, Green and Blue, each with a price of 0. Size will have a default value of « M », and possible options will be S (price 0), M (price 0), L (price 2), XL (price 5) and XXL (price 10).

Multiprice

data type string

ezmultiprice

Expected input

Fields are separated by a pipe.

The first item is the VAT Type, as a numerical value, starting from 1, as listed in the VAT management view.

The second item is the price type, 1 being inc. VAT, 0 being exc. VAT.

All subsequent items are grouped by 3, each set being the currency code (EUR, USD... matching the values in currency management), the price value, and the price type (AUTO or CUSTOM).

AUTO prices are NOT computed automatically based on the CUSTOM ones. This can easily be performed by calling updateAutoPriceList on the content object's attribute before publishing.

Example

Input

2|1|EUR|30|CUSTOM

Result

The multiprice attribute will be use the 2nd VAT item, and will be inc. VAT. The price will be set in Euros, with a price of 30, and a custom value type. When auto prices are updated, other AUTO values will be updated correctly.

Price

data type string

ezprice

Expected input

Three items, separated by pipes:
  • the actual price, as a float
  • VAT type, as declared in VAT management, starting from 1
  • price type, excl. (0) or incl. VAT (1)

Example

Input

::
30|1|1

Result

A price attribute with the standard VAT (1st one), including VAT (1), and a value of 30 XX.

Range option

data type string

ezrangeoption

Expected input

The method expects four items, separated by pipes:
  • The range option name, as a string
  • the range option start value, as integer
  • the range option end value, as integer
  • the range option step value, as integer

Example

Input

My range|1|5|1

Result

A range option named "My range", going from 1 to 5 by steps of 1.

User account

data type string

ezuseraccount

Expected input

from two to four items, separated by pipes:
  • login
  • email
  • password hash (optional)
  • password has type, as a string: md5_password, md5_user, md5_site, mysql, plaintext or crypt

Example

Input

johndoe|john@doe.com

Result

Creates a user named johndoe, with the email address john@doe.com
